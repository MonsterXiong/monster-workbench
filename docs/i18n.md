# 国际化 (i18n) 全量规范

> 本文档约束项目多语言国际化的文案管理、词典同步、组件接入与状态解耦策略。

---

## 1. 中文文案基本规范

用户可见文案统一简体中文，软件名称 `Monster Tools` 可保留英文。

要求：

- 文案短、准、直接
- 不展示解释产品是什么的废话
- 不混用中英文双标题
- 按钮优先使用动作词，例如"保存""新建""检查更新"

---

## 2. 文案统一管理与 useI18n 国际化接入

- **拒绝硬编码中文**：对于需要对齐国际化标准的页面，禁止在 template 或 script 中直接硬编码中文字符串。所有用户可见的业务文本（包括标题、提示信息、按钮标签、占位符等）应当统一管理。
- **词典层级同步与全扫描检查**：中英文多语言词典分别在 `src/locales/zh-CN.ts` 和 `en-US.ts` 中维护。新加入的词条必须在两个文件中以相同的多级路径（如 `system.title`）进行严格对称声明，以防在切换语言时因为 Key 缺失 fallback 失败，并在页面上直接退化显示 Raw Key（如 `tools.base64.clearBtn`）。**AI 在新增或重构组件翻译（使用 `t('...')` 等）时，必须全量扫描两个语言包，确保所用 Key 确实已在这两个语言包文件中完整定义**。
- **useI18n 引入原则**：在需要国际化文案的页面中，使用 `const { t } = useI18n()` 组合式函数获取翻译接口。在组件模板中使用 `{{ t('system.title') }}` 进行动态渲染，保障业务文案的统一提取与维护。
- **紧凑型表格与双栏对称排版标准**：在左右双栏控制台场景下，`BaseTable` 的字号及 padding 必须采用 `size="sm"` 紧凑模式，并结合包裹容器的 `h-full overflow-auto` 限制高度，确保大量数据加载时只在表格内部产生滚动，坚决防范破坏排版的纵向撑开变形。

---

## 3. 动态配置数据的全生命周期多语言规范

- **响应式表头列配置 (Columns)**：当在页面或组件中使用表格组件时，其表头 `columns` 定义数组如果包含中文标题，**必须**使用 Vue 的 `computed` 属性进行包装（例如 `const portColumns = computed(() => [ ... ])`），并在内部使用 `t()` 函数进行字段映射。禁止在 `script setup` 中将其声明为普通的静态常量，否则会导致用户在偏好中切换语言时，表头文字发生 fallback 滞后或无法同步翻译的低端体验。
- **弹窗及轻提示内联参数替换**：对于二次确认弹窗（Confirm Modal）中的多语言模版字串（如"确定强杀 PID 为 {pid} 的进程吗？"），应在语言包中统一保留参数插槽 `{pid}`，并在业务调用时通过 `.replace('{pid}', String(pid))` 等手段进行内联数据填充。禁止在代码中硬编码任何中文字符串拼装行为。

---

## 4. 全局公共选择器的多语言热更新规范

- **默认 placeholder 的延迟兜底翻译**：对于全局公共组件（例如 `AppPathSelector.vue`），若其属性（如 `placeholder`）包含默认中文，**绝对禁止**直接在 `withDefaults` 的 props 定义中硬编码中文字符串。
- **模板内联判定机制**：在 props 定义中应将 `placeholder` 默认值设为空字串 `""`，并在 `<template>` 的输入框绑定中，使用逻辑或操作符进行动态兜底（如 `:placeholder="placeholder || t('common.pathSelector.placeholder')"`）。此举能够确保在多语言环境热切换时，占位提示文本可以无缝刷新，没有任何滞后。
- **错误捕获及轻提示的 i18n 覆盖**：在组件内部触发 `triggerToast` 或是 `reject(new Error(...))` 时，所有的警告、异常文字（如"选择目录失败"）必须收拢到全局 `common.pathSelector` 等多语言词典中，统一通过 `t()` 接口进行调度，确保零中文裸露。

---

## 5. 分页组件的国际化重构规范

- **字符模板化配置**：对于在列表中频繁出现的分页组件（如 `NavigationPagination.vue`），其"共 X 条"、"上一页"、"下一页"等文本禁止硬编码。
- **数字参数的内联插槽置换**：文案（如"共 {total} 条"）必须使用插槽模板，在英文环境下映射为 `{total} items`。在组件模版中，必须通过 `t("navigation.pagination.total").replace("{total}", String(total))` 进行动态插值，确保在不同语言语系下单词复数及语序的正常排版。

---

## 6. 业务硬编码错误的布尔状态化解耦规范

- **杜绝特定语言字符串匹配**：在业务逻辑中，如果需要根据异常情况控制视图渲染（例如：只在非报错/转换成功时显示"复制"按钮），**绝对禁止**通过中文或英文文本匹配来进行条件过滤（如 `v-if="!output.startsWith('无效')"`），这会导致在语言切换后条件失效而产生严重的界面渲染 Bug。
- **强制引入布尔状态哨兵（State Flags）**：应当显式定义布尔类型的状态变量（如 `hasTsError = ref(false)`、`hasDateError = ref(false)`），在数据处理函数的 `catch` 块中进行赋值。在 HTML 模板中使用布尔值进行条件判断（如 `v-if="output && !hasDateError"`），实现核心业务逻辑与呈现文本的彻底解耦。
