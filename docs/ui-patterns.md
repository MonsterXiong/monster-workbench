# 弹窗、更新、目录遍历与数据一致性规范

> 本文档约束自动更新流程、Modal 弹窗收纳、目录遍历安全保护与文件-数据库关联一致性设计。

---

## 1. Tauri 客户端原生自动更新流程

- **非阻塞式静默检查**：后台自动轮询（默认每小时一次）必须是静默的。严禁在后台自动运行期直接弹出下载进度条或进行强制升级。
- **状态徽标（红点）与手动唤醒**：检查到新版本后，仅在左侧侧边栏版本号区域亮起红点标记。当用户手动点击版本号或检测到可用更新时，才调起更新确认弹窗。
- **日志渲染与更新确认**：弹窗内必须优雅排版展示 Release Notes，提供"稍后更新"与"立即升级"两个明确选项。点击立即升级后，弹窗就地转化为实时下载容量与百分比进度条状态，并在完成安装后自动调起 `relaunch()` 重启软件。
- **单例定时器保护**：由于更新 store 通常挂载于 Layout 级生命周期中，其 `initAutoCheck` 定时轮询接口必须使用局部状态锁（如 `autoCheckInitialized` 哨兵锁）进行防重入控制，防止热重启或路由切换时导致后台定时器重复创建与内存溢出。

---

## 2. 功能性二级配置弹窗 (Modal) 收纳原则

- **拒绝主视窗堆砌**：对于不影响核心操作流的辅助参数配置（如忽略目录列表、过滤阈值、递归层数上限等表单输入），禁止直接平铺摆放在主窗口区域。应收纳到二级配置弹窗中，主界面仅保留明确的设置入口。
- **遮罩与过渡动画**：弹窗遮罩层统一使用半透明且带高斯模糊的样式（如 `bg-slate-900/60 backdrop-blur-sm`），内部 Card 的入场与离场配合 Vue `<transition>` 的缩放和不透明度缓动动效（如 `scale-95 opacity-0` -> `scale-100 opacity-100`）。

---

## 3. 物理文件目录遍历防挂起拦截

- **排除特大文件夹递归**：由于前端直接读取或 Rust 遍历大体积工程目录（如包含 `node_modules`、`.git`、`dist` 的目录）可能因文件过多引起进程崩溃，必须在遍历入口处提供排除机制。对于匹配排除列表的子目录，仅在树形图中输出该文件夹本身，直接跳过并拦截向其子项进行更深度的遍历。
- **强制限制最大递归级数**：任何递归遍历目录结构的代码，必须强制附带最大递归层级保护参数（默认上限为 `10` 级，极限保护深度不得超过 `100` 级），在递归层数溢出时立刻切断并返回，绝对防止因循环软链接或深层嵌套引发的死循环与栈溢出。

---

## 4. 文件与数据库关联一致性与自愈设计原则

为了避免物理文件删除导致数据库中残留无效相对路径，进而在前端页面上渲染出破图或死链，必须落实以下规范：

- **删除前置引用检测**：任何物理文件、二进制图片在通过前端或底座进行物理擦除前，**必须首先**查询关联数据库表（如 `navigation`），确认是否仍有记录引用该文件的相对路径（如作为 `logo_path`、`bg_path` 等）。
- **用户提示与强制确认**：若检测到文件正被数据库引用，必须向用户弹出具有明确警示信息的二次确认弹窗（如 `ConfirmModal`），将关联了该文件的记录名称清晰罗列，由用户决定是否强行解绑并删除。
- **关联路径自愈与数据库同步**：一旦用户确认强制删除，系统在执行底座物理删除文件之前，必须先通过 Rust Command / Service 清空数据库中所有引用该文件的路径字段（设置为 `NULL` 或空字串），彻底消除数据死链。
