# Monster Workbench Style Guide

## 目标

- 现代化桌面软件风格
- 简约、美观、大气
- 全局统一，不做页面级各自发挥
- 优先保证中文字体清晰度和信息密度

## 整体布局

- 固定为左侧导航 + 中间工作区 + 可选底部状态栏
- 中间工作区不设置顶部说明横幅，页面直接进入工作状态
- 桌面全屏场景不使用 `max-width`
- 优先为桌面大屏设计紧凑布局，窗口缩小时再做适配

## 导航

- 左侧导航按“时间胶囊”思路处理，强调入口层级和当前状态
- 支持收起 / 展开
- 收起后仍能通过图标识别
- 当前项目一级导航固定为：`笔记`、`能力测试`、`设置`

## 视觉语言

- 使用克制的边框、阴影和圆角
- 卡片统一为低圆角、浅阴影、清晰边界
- 不使用营销式大标题、渐变英雄区、玻璃拟态或氛围光斑
- 所有页使用一致的卡片、表单、按钮体系

## 字体

- 系统字体栈：
  `system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, "Microsoft YaHei", sans-serif`
- 保持 Windows WebView2 下中文 ClearType 清晰
- 不引入未必要的在线字体

## 组件

- 统一采用 DaisyUI 5 基础类
- 图标统一使用 `lucide-vue-next`
- 输入框、文本域必须具备明显的填充背景、边框和焦点态
- 主要操作按钮风格统一，不同页面不重复发明控件样式

## 文案

- 用户可见内容统一简体中文
- 软件名称 `Monster Tools` 保留英文
- 文案尽量短，不解释常识，不重复描述
- 页面内容只保留与当前操作有关的信息

## 禁止项

- 禁止页面各自定义一套按钮、输入框和卡片视觉
- 禁止使用过细字号作为主要阅读字号
- 禁止强模糊、重渐变、重装饰背景
- 禁止为追求“科技感”牺牲中文可读性
